# 1강. 프로그래밍 언어 소개

## 프로그래밍 언어란 무엇인가?

- 주로 사람의 생각을 기계에 전달하는 데 사용
- 컴퓨터가 수행할 수 있고 사람이 읽을 수 있는 형태

## 프로그래밍 언어의 기능

- 기본 기능 - 작성력, 가독성, 실행 가능성
- 부가 기능 - 추상화, 모듈화
- 특성 - 기계적, 구조적, 가변적
- 고급 언어, 저급 언어

## 프로그래밍 언어의 구성 요소

- 데이터, 연산, 명령어, 서브프로그램, 타입, 모듈

# 2강. 프로그래밍 언어의 발전 및 동작 원리

## 프로그래밍 언어의 발전

- 아이디어 시대(튜링기계) → 전자식 등장(애니악, 콜로서스) → 저장 방식의 컴퓨터(애니박)
- 일괄 → 시분할 → DOS → GUI

## 프로그램 동작 원리

- 인출 → 해석 → 실행 주기 반복
- 메모리에 적재된 프로그램의 명령어를 수행
- 기계어(이진수) / 어셈블리어(CPU 종속) / 프로그래밍 언어
- 인터프리터는CPU의인출-해석-실행주기를흉내냄
- 인터프리터가하는해석과정을미리모두수행(효율적)
- 하이브리드 - 인터프리터방식과컴파일러방식을조합한방식(JVM)

## 프로그래밍 언어의 평가 기준

- 프로그래밍 언어의 요구사항 - 표현 풍부성, 유지 보수성, 실행 가능성
- 프로그래밍 언어 설계 원칙 - 규칙성(일반성, 직교성, 일관성), 추상화 지원, 복잡도 제어

## 여러 패러다임의 프로그램 예

- 명령형 - 프로그램을 일련의명령어나열로간주
- 절차형 - 프로그램을서브루틴이라는절차의집합으로구성하는방법, 재귀호출을사용하여프로시저를간단하게정의
- 구조화 - goto없이프로그램을작성하는방법, 블록과 서브루틴 활용, 현재 우리가 사용하는 대부분의 언어는 블록 구조
- 객체지향 - 프로그램은서로통신할수있는객체의집합, 재사용
- 함수형 - 값 자체에 집중, 프로그램수행이란원하는값을구하는것, 데이터는값으로,명령어는함수로취급
- 선언형 - 프로그램은논리식집합으로구성
